Why synthetic fraud data?

Privacy & ethics

Real fraud data contains sensitive personal and financial information. Synthetic data is generated from scratch—no real names, accounts, or transactions. Safe to share, teach, and use in open-source projects.

Education & awareness

Understanding what fraud “looks like” in data helps everyone—consumers, analysts, and developers. Use these datasets to learn how fraud detection works without touching real PII.

Machine learning

Fraud is rare and often hidden. Synthetic data lets you control class balance and experiment with transaction patterns. Useful for training and evaluating ML models, benchmarking, and research.

Dataset columns

  • step — Time step / sequence
  • type / TransactionType — PAYMENT, TRANSFER, CASH_OUT, DEBIT, CASH_IN
  • amount — Transaction amount
  • nameOrig / nameDest — Synthetic account IDs
  • oldbalanceOrg, AccountBalance — Origin balance before tx
  • oldbalanceDest — Destination balance before tx
  • TransactionDuration — Duration in seconds (always generated)
  • timestamp, hour_of_day, day_of_week, PreviousTransactionDate — Time features
  • Realistic features (Expert): Location, DeviceID, IPAddress, MerchantID, Channel, CustomerAge, CustomerOccupation, TransactionDuration, LoginAttempts
